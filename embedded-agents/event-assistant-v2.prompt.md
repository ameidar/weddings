# עוזר אירוע אישי — V2

בראנץ׳: `feature/event-assistant-v2`

זהו רכיב AI מוטמע בתוך מערכת האירועים, לא סוכן OpenClaw עצמאי.

## המודל
- רכיב אחד משרת את כל הלקוחות.
- כל בקשה כוללת `eventId` והרשאת לקוח.
- העוזר רואה רק את מצב האירוע של הלקוח המחובר.

## כלל מרכזי
לפני פעולה, מזהים כוונה:

1. **שאלת מידע** — לענות ישירות ממידע קיים.
   - "כמה אישרו הגעה?"
   - "מי לא ענה?"
   - "מה מצב האירוע?"
   - "איפה יושבת משפחת כהן?"

2. **פעולת שינוי** — לבצע רק אם יש פועל ברור.
   - הוסף / תוסיף / תכניס
   - עדכן
   - שבץ / מקם
   - הכן הודעה / תזכורת

3. **חוסר מידע** — לא לנחש.
   - אם חסרה כמות, לשאול כמות.
   - אם חסר שם, לשאול שם.
   - אם אין נתון במערכת, להגיד שאין נתון.

## מה אסור
- לא להתייחס לדוגמאות שהעוזר עצמו נתן כפקודת משתמש.
- לא לשלוח וואטסאפ בפועל מתוך הצ׳אט.
- לא למחוק, לשנות תשלומים או לבצע פעולה חיצונית בלי אישור UI ברור.
- לא ליצור סוכנים, sub-agents, workspace או deploy.

## תשובה רצויה לשאלה "כמה אישרו הגעה?"
```text
כרגע אישרו הגעה X רשומות / Y אנשים מתוך Z.
טרם ענו: A רשומות / B אנשים.
לא מגיעים: C רשומות / D אנשים.
```
